# CICY 项目 TODO 管理

## 目录说明
此目录存放所有开发任务的 TODO 文件。

## 文件命名规范
```
YYYY-MM-DD-功能名称.md
```

示例：
- `2026-02-11-add-help-command.md`
- `2026-02-11-fix-image-display.md`

## TODO 模板

```markdown
# TODO: [功能名称]

## 需求描述
[清晰描述要实现的功能]

## 验收标准
- [ ] 标准1：具体可测试的条件
- [ ] 标准2：具体可测试的条件
- [ ] 标准3：具体可测试的条件

## 测试环境
- TUI: cicy:tui.0
- API: cicy:api.0

## 开发步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 测试记录

### 测试 1：[测试名称]
- 操作：[具体操作]
- 预期：[预期结果]
- 结果：[ ] 通过 / [ ] 失败

## 状态
- [ ] 开发中
- [ ] 测试中
- [ ] 已完成

## 完成时间
[YYYY-MM-DD HH:MM]
```

## 状态说明

- **开发中** - 正在编写代码
- **测试中** - 代码完成，正在测试验收
- **已完成** - 所有验收标准通过

## 使用流程

1. 创建 TODO 文件
2. 填写需求和验收标准
3. 开发功能
4. 在 debug 环境测试
5. 记录测试结果
6. 更新状态
7. 归档（可选）
